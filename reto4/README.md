# Reto 4 - Análisis de Números

## Descripción
Este reto consiste en crear un programa que reciba un número entero y realice múltiples validaciones matemáticas: determinar si es par o impar, positivo o negativo, y si es primo.

## Objetivos de Aprendizaje
- Implementar modularización mediante funciones
- Aplicar retorno de valores entre funciones
- Utilizar estructuras condicionales múltiples
- Implementar algoritmos matemáticos (validación de primos)
- Aplicar operadores aritméticos y de comparación
- Lógica de programación matemática

## Especificaciones
Crea un programa que permita:
- Recibir un número entero
- Determinar si es par o impar
- Determinar si es positivo, negativo o cero
- Determinar si es primo o no
- Mostrar un resumen completo con todas las validaciones

## Requisitos Técnicos
- Cada validación debe estar en una función diferente
- Las funciones deben comunicarse mediante retorno de valores
- Implementación correcta del algoritmo de números primos
- Manejo de casos especiales (0, 1, números negativos)
- El código debe ser modular y reutilizable

## Criterios de Evaluación
- Correcta implementación de las funciones requeridas
- Precisión en las validaciones matemáticas
- Manejo correcto de casos especiales
- Uso apropiado de condicionales
- Legibilidad del código
- Cumplimiento de los criterios de aceptación

## Consideraciones Matemáticas
- El número 0 se considera par, no es positivo ni negativo, y no es primo
- El número 1 no se considera primo por definición matemática
- Los números negativos generalmente no se consideran primos
- Un número primo es mayor que 1 y solo divisible por 1 y por sí mismo
