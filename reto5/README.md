# Reto 5 - Reto Creativo

## Descripción
Este es un reto creativo donde debes proponer y desarrollar tu propio mini-sistema aplicando todos los conceptos aprendidos sobre funciones, modularización y buenas prácticas de programación.

## Objetivos de Aprendizaje
- Aplicar de forma integral todos los conceptos aprendidos
- Desarrollar habilidades de diseño de software
- Implementar soluciones creativas y funcionales
- Demostrar comprensión profunda de modularización
- Practicar buenas prácticas de programación

## Requisitos del Reto

### Conceptos a Aplicar (Obligatorios)
- **Funciones tradicionales y funciones flecha**: Debes incluir ambos tipos
- **Parámetros y argumentos**: Todas las funciones deben tener parámetros apropiados
- **Retorno de valores**: Las funciones deben comunicarse mediante retorno de valores
- **Modularización**: Cada funcionalidad en una función diferente
- **Buenas prácticas**: Código legible, organizado y mantenible

### Sugerencias de Proyectos
Puedes desarrollar cualquiera de estos mini-sistemas o proponer uno propio:
- Calculadora avanzada
- Registro de estudiantes
- Agenda de contactos
- Simulador de préstamos
- Sistema de inventario
- Juego de adivinanza
- Conversor de monedas
- Gestor de tareas
- ¡Cualquier otra idea que tengas!

## Criterios de Aceptación

### Funcionales
- El proyecto debe tener una funcionalidad clara y útil
- Debe incluir al menos 5-7 funciones diferentes
- Debe usar tanto funciones tradicionales como funciones flecha
- Todas las funciones deben tener parámetros y retornar valores apropiados

### Técnicos
- Código modular y bien organizado
- Nombres descriptivos para funciones y variables
- Manejo apropiado de errores y validaciones
- Funcionalidad completa y probada

### Calidad
- Código legible y mantenible
- Aplicación de buenas prácticas de programación
- Documentación básica del sistema

## Criterios de Evaluación
- Creatividad y originalidad del proyecto
- Correcta aplicación de todos los conceptos requeridos
- Calidad y organización del código
- Funcionalidad completa del sistema
- Documentación adecuada
- Demostración de comprensión de los temas

## Recomendaciones
- Elige un proyecto que te interese
- No te compliques demasiado - mejor simple y bien hecho
- Enfócate en la calidad del código
- Documenta brevemente cada función
- Prueba exhaustivamente tu sistema
- Diviértete desarrollando tu propia solución!
